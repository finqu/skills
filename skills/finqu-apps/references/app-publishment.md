# App Publishment

How to publish your Finqu partner app from draft to public.

## Publishment States

| State       | Visibility          | Requirements                      |
| ----------- | ------------------- | --------------------------------- |
| **Draft**   | Only you            | Default state when app is created |
| **Private** | Shareable via link  | Requires review                   |
| **Public**  | Listed in app store | Requires review                   |

## Process

1. **Draft** — Build and test your app. Not visible to merchants.
2. **Submit for review** — When ready, submit for Finqu team review.
3. **Private** — After approval, app is shareable via a direct link. Merchants can install via the link but won't find it in listings.
4. **Public** — After further review, app appears in the public app listing for all merchants.

## Review Requirements

To pass review, your app should:

- Have a clear description and purpose
- Implement the OAuth flow correctly
- Handle errors gracefully
- Follow Finqu's app terms and guidelines
- Request only necessary scopes
